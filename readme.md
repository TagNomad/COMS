# 产品目录与订单管理系统 功能需求及开发框架

## 项目概述

本项目旨在开发一套基于Web的 **产品目录与订单管理系统**，通过该系统实现对产品信息、客户信息和订单的全流程管理，并提供灵活的客户价格管理体系和销售统计报表功能。该系统将用于提升企业供应链管理效率，优化客户服务体验。

---

## 功能需求

### 1. **产品目录管理模块**
该模块包括对产品信息的管理，包括增、删、改、查功能。
#### 1.1 产品信息管理
- 功能描述：对产品的基本信息进行增、删、改、查。
- 主要操作：
  - 新增产品：录入产品名称、分类、规格、价格、状态等信息。
  - 修改产品信息：编辑已有产品的信息。
  - 删除产品：删除产品记录，并更新库存。
  - 查看产品：按产品名称、分类等条件查询产品信息。

#### 1.2 产品分类管理
- 功能描述：管理多级产品分类。
- 主要操作：
  - 添加分类：支持多级分类管理。
  - 修改分类：支持修改分类名称及父级分类。
  - 删除分类：支持删除分类，删除前需确认该分类下无产品。
  - 查看分类：支持按层级查看分类信息。

#### 1.3 产品库存管理
- 功能描述：对每个产品的库存进行管理。
- 主要操作：
  - 更新库存：每当有订单创建或产品入库时，更新库存数量。
  - 查询库存：查看产品当前的库存数量。

### 2. **客户管理模块**
该模块包括客户基本信息的管理，以及根据客户等级设定不同的产品价格。
#### 2.1 客户信息管理
- 功能描述：管理客户的基本信息。
- 主要操作：
  - 新增客户：录入客户名称、地址、联系人等信息。
  - 修改客户：编辑客户的基本信息。
  - 删除客户：删除客户记录，删除前需确保该客户无订单。
  - 查询客户：按条件查询客户信息。

#### 2.2 客户价格管理
- 功能描述：管理不同等级客户的产品价格。
- 主要操作：
  - 设置价格：为不同客户等级设置不同的产品价格。
  - 修改价格：根据需要修改客户等级对应的产品价格。
  - 查询价格：查询某个客户等级的产品价格。

### 3. **订单管理模块**
该模块用于创建和管理客户订单。
#### 3.1 订单创建
- 功能描述：为客户创建销售订单。
- 主要操作：
  - 创建订单：选择客户并添加产品到订单中。
  - 计算总价：自动计算订单的总金额。
  - 更新库存：根据订单生成时减少相应产品的库存。

#### 3.2 订单状态跟踪
- 功能描述：跟踪订单的状态。
- 主要操作：
  - 更新订单状态：修改订单状态（如待审核、已确认、已发货、已完成等）。
  - 查询订单状态：根据条件查询订单状态。

#### 3.3 订单查询
- 功能描述：根据不同条件查询订单信息。
- 主要操作：
  - 按订单编号、客户名称、订单日期等条件查询订单。

### 4. **销售统计与报表模块**
该模块用于生成销售数据报表，并提供销售统计信息。
#### 4.1 销售统计
- 功能描述：根据时间范围统计销售额、销售量等。
- 主要操作：
  - 按时间范围统计：生成销售统计报表，包括总销售额和总销售数量。

#### 4.2 热销产品排行
- 功能描述：统计销售的热销产品。
- 主要操作：
  - 按销售数量排序：查询并排序热销产品，为采购和营销决策提供依据。
